## 一、系统概览

桌面精灵是一个悬浮在 Web 应用上的虚拟陪伴角色，贯穿用户从首次注册到日常创作的全生命周期。核心功能包括：新用户引导、孵化选择、日常互动（签到/聊天/道具）、等级成长、AI 反馈、空闲动画系统。

### 生命周期

```
新用户注册 → 自动创建精灵蛋(L0) → 10步互动引导 → 孵化动画 → L1精灵
    ↓
日常使用：签到 → 升级(L1→L9) → 精灵豆系统 → 道具购买/使用 → AI聊天 → 空闲动画循环
```

## 三、等级与数值体系

### 3.1 升级所需天数

| 等级 | 所需天数 | 说明 |
|------|----------|------|
| L1 | 0 | 初始等级（孵化后） |
| L2 | 26 | 约 1 个月 |
| L3 | 58 | 约 2 个月 |
| L4 | 96 | 约 3 个月 |
| L5 | 140 | 约 4.5 个月 |
| L6 | 190 | 约 6 个月 |
| L7 | 245 | 约 8 个月 |
| L8 | 305 | 约 10 个月 |
| L9 | 365 | 约 1 年（满级） |

计算公式：`currentLevel = max(lvl) where totalDays >= LEVEL_DAYS[lvl]`
`totalDays = totalActiveDays + bonusDays`

### 3.2 精灵豆经济

- **来源**：每日签到自动获得（后端逻辑）
- **消费**：购买道具（消耗精灵豆）
- **兑换**：累计消费每 100 豆可兑换约 1.45 天付费时长（`floor(totalBeanSpent / 100 / 0.69) - convertedDays`）

#### 渲染分支

组件有两层渲染分支：

1. **无精灵记录**（`!status?.hasSprite`）：渲染加载占位符（红色脉冲圆），此阶段 Hooks 已全部执行。
2. **有精灵记录**（`status?.hasSprite`）：
   - 先执行 hydration guard：`if (!mounted) return null;`
   - 再计算 `isEgg` / `debugSrc`
   - 渲染精灵蛋或已孵化精灵 + 所有子组件

#### 交互行为

| 交互方式 | L0 引导期间 | L0 引导后/已孵化 |
|----------|------------|-----------------|
| 拖拽 | 禁用 | 自由拖拽（鼠标+触摸） |
| 双击 | 提示"先完成引导" | 随机鼓励文案 |
| 右键 | 提示"先完成引导" | 未孵化 → HatchDialog；已孵化 → ContextMenu |

| 图片 mouseup | - | 恢复空闲动画 + 结束拖拽 |

**交互引导（临时）**：
- L0 精灵蛋用户 **5 秒无任何操作** 后，在精灵右侧显示引导气泡
- 气泡内容：双击（破壳孵化）、右键（选择孵化方向）、拖动（移动位置）
- 用户执行任意操作（双击/右键/拖动）后立即消失
- **仅提示一次**，通过 `localStorage('sprite-interaction-hint-shown')` 持久化，刷新页面不再显示

#### 空闲动画管理

精灵图片 `<img>` 标签从 `sprite-manifest.json` 读取路径，根据当前等级自动分配动画类：

```typescript
// 动画池 key 映射
L0:        L0 精灵蛋阶段
L1-L2:     等级 1-2（sway, sleep-l12）
L3-L5:     等级 3-5（tilt, bounce, sleep-l12）
L6-L9:     等级 6-9（float, prop-play, sleep-l69）
```

管理器特性：
- **随机切换**：每 3-8 秒随机切换动画，不重复上一次动画（如果动画池 > 1）
- **疲劳感知**：睡眠动画通过 `prefers-reduced-motion` 媒体查询自动禁用
- **拖拽联动**：拖拽时暂停动画，释放后恢复
- **等级变更**：自动销毁旧实例，创建新实例
- **组件卸载**：自动清理定时器和 DOM 类

#### 图片显示规范

**容器尺寸**：
- `SPRITE_SIZE = 160px`（外层容器固定 160×160px）
- 图片使用 `width: 100%, height: 100%, objectFit: contain` 自适应容器

**原始尺寸规范**（设计/输出标准，非显示尺寸）：
- L0-L4: 256×256px（精灵蛋 + 低等级精灵）
- L5-L9: 512×512px（高等级精灵，细节更丰富）
- 所有图片在 160×160 容器内等比缩放，保持宽高比，不拉伸变形

**后续资产一致性要求**：
- 新增精灵种类（animal/fox, element/wind）必须遵循相同尺寸规范
- 每个等级 L0-L9 都需提供对应图片，缺失时回退到 universal_egg
- 图片导出格式：PNG，透明背景
- 图片命名：`{species}_L{level}.png`（如 `sunflower_L3.png`）
- 存放路径：`/public/assets/sprites/{species}/{variant}/`
- 通用精灵蛋：`/public/assets/sprites/common/universal_egg.png`
- 所有新资产更新后需同步更新 `sprite-manifest.json` 路径映射

#### 图片加载

```typescript
// debugSrc 计算逻辑（hydration guard 之后）
const debugSrc = !status?.isHatched
  ? spriteManifest.universal_egg
  : (spriteManifest.sprites[status.species]?.[status.variant]?.images[`L${status.level}`]

- L0（未孵化）使用 `universal_egg`
- L1+ 使用 `sprites[species][variant].images[L{level}]`

## 八、关键交互流程

### 8.1 新用户首次登录

```
1. 用户登录 → SpriteComponent 挂载
2. getSpriteStatus 返回 { hasSprite: false }
3. 自动调用 createEgg → 创建精灵蛋记录
4. 状态刷新 → isL0Guide = true, guideStep = 0
5. 自动激活引导 (setGuideActive = true)
6. 显示精灵蛋（🥚 + wobble 动画）
7. 精灵蛋气泡显示 Step 0 文案
8. GuideOverlay 渲染 Step 0 引导卡片（居中）
```

### 8.2 L0 引导逐步推进

```
用户点击"下一步" → handleL0GuideNext()
  ↓
检查下一步是否需要导航（如 step 2 → /project/new）
  ↓
如果需要导航 → window.location.href = navigateTo
  ↓
调用 advanceGuide mutation → guideStep + 1
  ↓
GuideOverlay 根据新 step 重新渲染
  ↓
... 逐步走完 0 → 9 ...
```

### 8.3 引导完成 → 孵化

```
Step 9 点击"开始孵化" → handleL0GuideNext()
  ↓
nextStep = 10 → claimGuideReward mutation
  ↓
后端：bonusDays += 3, isHatched = true, level = 1
  ↓
setGuideActive = false
  ↓
500ms 后 → setShowHatchDialog = true
  ↓
HatchDialog 以 mode="guided" 打开
  ↓
用户选择系别/种类/名字/称呼/风格
  ↓
点击"开始孵化" → 显示 HatchAnimation
  ↓
8.5s 后动画完成 → hatch mutation 提交后端
  ↓
后端：设置 species/variant/name，isHatched=true, guideStep=10
  ↓
refetch() → 精灵组件重新渲染为已孵化状态
  ↓
IdleAnimationManager 初始化 → 开始空闲动画循环
```

### 8.4 右键菜单孵化（跳过引导）

```
L0 精灵蛋（引导未激活时）右键点击 → handleContextMenu
  ↓

  ↓
HatchDialog 打开（无 HatchAnimation）
  ↓
用户选择系别/种类/名字/称呼/风格
  ↓

  ↓
后端：设置 species/variant/name，isHatched=true, guideStep=10
  ↓
refetch() → 精灵组件重新渲染为已孵化状态

### 8.5 每日签到 + AI 反馈

```
页面加载 → status 返回
  ↓
已孵化 → 自动调用 checkin mutation
  ↓
checkin 返回 { alreadyCheckedIn: true } → 今日已签到
  ↓

  ↓

  ↓

### 8.6 道具购买 → 使用

  ↓

  ↓

```

  ↓

## 十、已知问题与注意事项

### 10.1 Hydration 问题

`GuideOverlay` 使用 `mounted` 状态确保只在客户端 hydration 完成后才渲染，避免 SSR/CSR 不一致导致的 React hydration 错误。`SpriteComponent` 同样使用 `mounted` + hydration guard（`if (!mounted) return null`）确保所有 Hooks 在首次渲染时一致执行。

### 10.2 旧版 L1 引导兼容

`GuideDialog` 组件仍存在于代码中，适用于 `isHatched=true && guideStep 1-4` 的已有用户。新用户使用 `GuideOverlay` 系统（step 0-9）。两套系统并行运行但互不干扰。

### 10.3 导航跳转限制

Step 4-7（项目页面相关）需要具体的 `projectId`，但 `L0_GUIDE_STEPS` 中 `navigateTo` 设为 `null`。当前依赖用户自己创建项目后手动导航到这些页面。如需自动跳转，需要后端返回用户最近编辑的项目 ID。

### 10.4 形象加载优先级

```
sprite-manifest.json → art_assets (已发布) → sprite_images (旧表) → universal_egg
```

### 10.5 精灵豆数值

- 兑换公式：`floor(totalBeanSpent / 100 / 0.69) - convertedDays`
- 这意味着每消费约 69 精灵豆可兑换 1 天付费时长

### 10.6 精灵图片资源

- 精灵图片需提前部署到 `/public/assets/sprites/` 目录
- 当前仅 plant/sunflower 有完整 L0-L9 图片映射，animal/fox 和 element/wind 的 images 为空，使用 universal_egg 回退

### 10.7 精灵容器尺寸

- `SPRITE_SIZE = 160px`（初始值 72px，已调整为 160px）
- 图片使用 `objectFit: contain` 在容器内等比缩放
- 原始图片尺寸：L0-L4 为 256×256px，L5-L9 为 512×512px
- 所有新资产必须遵循相同尺寸规范

### 10.8 交互引导

- 交互引导气泡仅在 L0 精灵蛋状态下，用户 5 秒无操作后触发
- 通过 `localStorage('sprite-interaction-hint-shown')` 持久化，仅提示一次
- 用户双击/右键/拖动任意操作后立即消失

### 10.9 升级测试

- ContextMenu 中的"升级测试 ✨"按钮仅 `level < 9` 时显示
- 点击后调用 `testLevelUp` mutation，通过增加 `bonusDays` 触发等级变化
- 升级动画使用 `LevelUpAnimation` 组件（4 阶段：glow → transform → reveal → done）

### 10.10 资产输出标准

为确保后续新增精灵资产与已导入资产保持一致，制定以下输出标准：

| 规范项 | 要求 |
|--------|------|
| 图片格式 | PNG，透明背景 |
| 原始尺寸 | L0-L4: 256×256px；L5-L9: 512×512px |
| 显示尺寸 | 统一在 160×160px 容器内 `objectFit: contain` 等比缩放 |
| 命名规则 | `{species}_L{level}.png`（如 `sunflower_L3.png`） |
| 精灵蛋 | `universal_egg.png`（所有系别通用） |
| 存放路径 | `/public/assets/sprites/{species}/{variant}/` |
| 清单更新 | 新增图片后必须更新 `sprite-manifest.json` 路径映射和动画池配置 |
| 动画池 | 每个精灵种类需在 `animation_pools` 中定义各等级对应的可用动画 |

### 10.11 Debug 日志

当前 `sprite-component.tsx` 包含调试日志（`console.log('[Sprite Debug] ...')`），用于排查图片加载问题。确认图片加载正常后可移除。
