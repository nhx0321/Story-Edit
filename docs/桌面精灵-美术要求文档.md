# 桌面精灵·美术要求文档

> Story Edit SaaS 平台 — 精灵形象美术规范
>
> 创建日期：2026-04-14

---

## 一、概述

桌面精灵是 Story Edit SaaS 平台的虚拟陪伴角色，贯穿用户从注册到日常创作的全生命周期。精灵分为三个系别，每个系别有独特的视觉风格，通过 L0（精灵蛋）到 L9（满级）共 10 个等级逐步成长变化。

---

## 二、精灵系别

| 系别 | 种类 | 视觉风格 | 配色倾向 |
|------|------|----------|----------|
| 植物系 (plant) | 向日葵 (sunflower) | 自然、温暖、可爱 | 绿色 + 黄色 |
| 动物系 (animal) | 小狐狸 (fox) | 灵动、活泼、亲近 | 橙色 + 白色 |
| 元素系 (element) | 小风灵 (wind) | 空灵、透明、飘逸 | 青色 + 半透明白 |

---

## 三、技术规格

### 3.1 图片格式

| 属性 | 要求 |
|------|------|
| 格式 | PNG（24 位，支持透明度） |
| 背景 | **透明背景**，不得有白色或其他颜色底色 |
| 色彩空间 | sRGB |
| 压缩 | 无损压缩，保留最大细节 |

### 3.2 尺寸规范

#### 原始输出尺寸

| 等级范围 | 画布尺寸 | 说明 |
|----------|----------|------|
| L0（精灵蛋） | 256×256px | 通用精灵蛋，所有系别共用 |
| L1-L4 | 256×256px | 低等级精灵，造型简单 |
| L5-L9 | 512×512px | 高等级精灵，细节丰富 |

#### 显示尺寸

- 所有精灵形象在 Web 端统一显示为 **160×160px**
- 使用 `objectFit: contain` 等比缩放，不拉伸变形
- 原始尺寸 > 160px 时自动缩小，< 160px 时保持原始大小居中显示

#### 安全区域

- 精灵形象主体应控制在画布中心 **80% 区域** 内
- 即 256px 画布：主体不超过 204×204px
- 即 512px 画布：主体不超过 409×409px
- 留白用于动画效果（呼吸、摇摆、浮动等）

### 3.3 命名规则

```
{species}_L{level}.png

示例：
  sunflower_L0.png    # 向日葵 L0 精灵蛋
  sunflower_L1.png    # 向日葵 L1
  sunflower_L5.png    # 向日葵 L5
  sunflower_L9.png    # 向日葵 L9（满级）

通用精灵蛋：
  universal_egg.png   # 所有系别共用
```

### 3.4 存放路径

```
/public/assets/sprites/{species}/{variant}/

示例：
  /public/assets/sprites/plant/sunflower/sunflower_L0.png
  /public/assets/sprites/plant/sunflower/sunflower_L1.png
  ...
  /public/assets/sprites/common/universal_egg.png
```

## 四、等级成长视觉要求

### 4.1 精灵蛋 (L0)

- 所有系别共用 `universal_egg.png`
- 造型：光滑蛋形，暖色调，暗示生命孕育
- 细节：可有轻微光泽/高光

### 4.2 等级递进原则

| 等级 | 视觉变化要求 |
|------|-------------|
| L1 | 刚孵化，幼体形态，造型简单可爱 |
| L2-L3 | 逐渐长大，增加少量细节（表情、小装饰） |
| L4-L5 | 中期成长，明显的外观变化，增加配饰/能力表现 |
| L6-L7 | 接近成熟，丰富的细节，特效元素（光环、粒子感） |
| L8-L9 | 完全体，华丽外观，满级标志（皇冠、光环等） |

### 4.3 设计建议

- **渐进式变化**：相邻等级之间应有明显的视觉差异，用户能一眼看出升级
- **特征保留**：每种精灵的核心识别特征（如向日葵的花盘、狐狸的尾巴）应在所有等级中保持一致
- **表情变化**：随等级提升，表情从呆萌 → 自信 → 成熟

## 五、动画适配要求

精灵形象需适配以下 CSS 动画效果：

| 动画名 | 效果描述 | 适用等级 |
|--------|---------|---------|
| egg-wobble | 左右微摆 + 上下浮动 | L0 精灵蛋 |
| sprite-idle | 缓慢呼吸（上下 6px + 微缩放） | 已孵化默认 |
| sway | 左右摇摆（±2deg） | L1-L2 |
| sleep-l12 | 缩小 + 微旋转（睡眠） | L1-L2 |
| tilt | 倾斜（±4deg） | L3-L5 |
| bounce | 弹跳（上下 6px） | L3-L5 |
| float | 浮动（上下 4px + 微缩放） | L6-L9 |
| prop-play | 摇摆 + 微扭曲 | L6-L9 |
| sleep-l69 | 缩小 + 微旋转（睡眠） | L6-L9 |
| pulse | 呼吸光效 | 通用 |

### 动画适配注意事项

1. **安全区域留白**：动画会导致元素超出原始边界，必须预留空间
2. **缩放兼容**：部分动画含 `scale(1.02~1.04)`，形象不能因微缩放而裁剪
3. **旋转兼容**：摇摆动画会旋转 ±2~4deg，形象重心应稳定
4. **透明背景**：必须透明，动画过程中不能出现方形底色

---

## 六、可访问性

- **减少动效**：支持 `prefers-reduced-motion` 媒体查询，动画会自动减弱
- **视觉对比**：精灵形象应有足够对比度，在白色和深色背景下均可辨识
- **尺寸适中**：160×160px 显示尺寸确保在小屏幕上仍可辨识

## 九、后续资产一致性

为确保未来新增精灵与现有资产保持一致：

1. **尺寸一致**：严格遵循 256px / 512px 分级规范
2. **格式一致**：PNG + 透明背景
3. **命名一致**：`{species}_L{level}.png` 格式
4. **路径一致**：存放在 `/public/assets/sprites/{species}/{variant}/`
5. **清单同步**：新增后更新 `sprite-manifest.json` 路径映射和动画池配置
6. **安全区域**：主体不超过画布 80%，预留动画空间
7. **视觉连贯**：相邻等级有明显差异，核心识别特征贯穿所有等级
