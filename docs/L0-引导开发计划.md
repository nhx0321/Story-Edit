# L0 精灵蛋引导教程·开发计划

## 概述

将引导从 L1 提前到 L0 精灵蛋阶段，设计 10 步互动式引导（步骤 0-9），引导完成后触发孵化动画进入 L1。

---

## Phase 1：后端改造（服务端）

### 1.1 新增「蛋模式」孵化接口

**文件**: `apps/server/src/services/sprite/router.ts`

- 新增 `createEgg` 接口：创建 L0 精灵蛋记录，不设置 species/variant，`isHatched=false`, `guideStep=0`
- 修改现有 `hatch` 接口：拆分为两个阶段
  - `createEgg` → 创建精灵蛋
  - `completeHatch` → 选择系别/命名 + 设置 `isHatched=true`, `level=1`

### 1.2 新增引导步骤推进接口

**文件**: `apps/server/src/services/sprite/router.ts`

- 修改 `advanceGuide`：支持 0-10 步骤（当前仅支持 1-4）
- 新增 `navigateGuideStep`：根据当前步骤返回推荐的目标页面路径

### 1.3 新增引导奖励发放

**文件**: `apps/server/src/services/sprite/router.ts`

- 修改 `claimGuideReward`：引导完成（step=10）时自动发放 +3 天

**涉及文件**:
- `apps/server/src/services/sprite/router.ts` — 修改/新增接口
- `apps/server/src/db/schema.ts` — 无需修改（guideStep 字段已满足 0-10 范围）

## Phase 5：精灵蛋 L0 形象

### 5.1 L0 精灵蛋显示

- 当前 `getStatus` 在 `isHatched=false` 时已显示精灵蛋
- 确认精灵蛋图片从 art_assets / sprite_images / 占位 SVG 正确加载
- L0 阶段禁用拖拽（引导期间固定位置或限制范围）

### 5.2 引导期间精灵蛋行为

| 行为 | 引导中 | 说明 |
|------|--------|------|
| 拖拽 | 禁用 | 避免干扰引导流程 |
| 双击 | 自定义 | 显示「请先完成引导哦~」 |
| 右键 | 禁用 | 避免引导流程被打断 |
| 气泡 | 引导文案 | 替代默认气泡 |

## 风险与注意事项

1. **引导过程中用户刷新页面**：需要从后端恢复 `guideStep` 状态，并跳转到对应页面
2. **引导期间用户创建多个项目**：引导 step 4-7 基于第一个项目进行，如果用户已有项目则跳过创建步骤
3. **元素选择器失效**：如果页面结构调整导致 `data-guide-target` 元素不存在，引导应自动跳过该步高亮，仅显示文案
4. **移动端适配**：引导遮罩和卡片需支持触摸事件
5. **引导跳过后的恢复**：下次登录时需要判断当前页面是否匹配 `guideStep` 对应的目标页面
