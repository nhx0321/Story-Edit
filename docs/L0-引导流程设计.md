# L0 精灵蛋·用户引导教程设计方案

## 背景

当前引导系统（`guideStep 1-4`）在精灵孵化（L1）后触发，使用纯文本对话框，引导内容过于简略，无法覆盖产品核心功能。

本方案将引导提前到 **L0 精灵蛋阶段**，设计一套 **互动式、分步骤、带高亮指引** 的完整新手教程，让用户在精灵蛋的陪伴下快速了解平台核心功能。引导完成后精灵破壳升到 L1。

---

## 一、引导流程总览

```
用户首次登录 → 自动获得精灵蛋(L0) → 引导教程开始(10步) → 孵化动画 → L1
```

### 引导步骤设计

| 步骤 | 场景 | 引导内容 | 用户操作 | 耗时 |
|------|------|---------|---------|------|
| 0 | 精灵蛋登场 | 精灵蛋出现在屏幕右下角，做轻微摇晃动画。弹出气泡：「你好呀！我是你的写作伙伴，先来完成一个小任务吧~」 | 点击「好的」开始引导 | ~5s |
| 1 | 认识工作台 | 高亮顶部导航栏，标注「工作台」「模板广场」「设置」功能。精灵蛋气泡：「上面是你的导航栏，可以在不同页面间切换哦」 | 点击「知道了」 | ~8s |
| 2 | 认识工作台(主区域) | 高亮工作台页面：项目列表、新建项目按钮、VIP状态。精灵蛋气泡：「这里是你所有项目的家，点击「新建项目」开始创作」 | 点击「新建项目」进入创建页 | ~15s |
| 3 | 创建第一个项目 | 进入项目创建页面，高亮表单字段（名称、题材、风格、方法论）。精灵蛋气泡：「填好信息后，你的第一个项目就创建好了」 | 填写并提交项目 | ~30s |
| 4 | 项目页·侧边栏导航 | 进入项目页，高亮左侧导航栏：概览、大纲、设定、正文、模板、AI助手。精灵蛋气泡：「左边是你的项目导航，每个模块都有不同的功能」 | 点击「知道了」 | ~10s |
| 5 | 项目页·概览页 | 高亮概览页的统计卡片（卷/单元/章节、字数、进度）和快捷操作。精灵蛋气泡：「这里可以看到你的创作进度，点击快捷操作快速进入编辑」 | 点击「知道了」 | ~10s |
| 6 | 大纲页面 | 高亮大纲页面的卷/单元/章节结构，标注编辑入口。精灵蛋气泡：「大纲是你故事的骨架，先规划好再写正文会事半功倍」 | 点击「知道了」 | ~10s |
| 7 | 编辑器页面 | 高亮编辑器的核心区域：正文编辑区、AI辅助面板、版本管理。精灵蛋气泡：「这里是你的主战场！写正文、用AI辅助、管理版本都在这里」 | 点击「知道了」 | ~15s |
| 8 | 模板广场 | 高亮模板广场，标注官方模板、打赏、导入功能。精灵蛋气泡：「没有灵感时，来模板广场逛逛吧，有很多实用的创作模板」 | 点击「知道了」 | ~10s |
| 9 | 精灵互动·回归 | 回到工作台，精灵蛋做摇晃动画。弹出气泡：「恭喜你学会了基本操作！双击我可以随时互动，右键可以打开菜单。现在我要破壳而出了！」→ 播放破壳孵化动画 | 点击「开始孵化」 | ~10s |

**总耗时约 2-3 分钟**，全部步骤为引导式而非强制，用户可随时跳过。

## 二、引导技术实现方案

### 2.1 引导引擎

使用**轻量引导引擎**（不引入第三方库），核心机制：

1. **步骤配置化**：每步定义 `target`（高亮元素选择器）、`title`、`text`、`action`（下一步行为）
2. **遮罩层 + 高亮**：半透明遮罩覆盖全屏，通过 `clip-path` 或 `box-shadow` 镂空目标元素
3. **引导卡片**：浮动在目标元素附近，显示标题、描述、操作按钮
4. **步骤状态持久化**：通过 `guideStep` 字段（0-10），后端 `user_sprites.guide_step` 存储
5. **精灵蛋伴随**：整个引导过程中精灵蛋始终可见，配合气泡对话

### 2.2 引导数据结构

```typescript
interface GuideStep {
  step: number;               // 0-9
  title: string;              // 标题
  text: string;               // 描述文本
  target?: string;            // CSS 选择器（要亮的元素）
  placement?: 'top' | 'bottom' | 'left' | 'right';  // 卡片位置
  action?: 'next' | 'click' | 'navigate';  // 按钮行为
  navigateTo?: string;        // action=navigate 时的目标路径
  waitForElement?: boolean;   // 是否等待 target 元素出现
  onComplete?: () => void;    // 步骤完成回调
}
```

### 2.3 引导触发逻辑

```
用户首次登录（isHatched=false && guideStep=0）
  → 自动创建精灵蛋记录（后端 hatch 接口增加 "egg-only" 模式）
  → 前端显示精灵蛋 + 引导气泡
  → 用户点击「好的」进入 step 1
  → 每完成一步 guideStep++
  → step 9 完成后触发孵化动画
  → 孵化完成后 isHatched=true, guideStep=10, level=1
```

### 2.4 精灵蛋 L0 特殊状态

| 状态 | 说明 |
|------|------|
| `isHatched=false` | 精灵蛋阶段 |
| `guideStep=0` | 等待引导开始 |
| `guideStep=1-9` | 引导进行中 |
| `guideStep=10` | 引导完成，等待孵化 |
| `isHatched=true, level=1` | 孵化完成，进入 L1 |

## 四、孵化动画设计（L0 → L1）

```
点击「开始孵化」
  → 精灵蛋从屏幕中央放大 1.5 倍
  → 蛋表面出现裂纹（CSS 动画，2-3 条裂纹）
  → 蛋开始摇晃（越来越剧烈）
  → 蛋从顶部裂开，白色光芒射出
  → 碎片四散飞落（粒子效果）
  → 新精灵从光芒中出现（系别对应形象）
  → 精灵做伸展动作 + 开心表情
  → 气泡：「我出来啦！以后请多关照~ {用户昵称}」
  → 显示系别/名称选择（复用现有 HatchDialog step 0-4）
  → 完成后进入 L1 状态
```

> **注意**：当前孵化流程是「先选择系别→命名→孵化」，新流程改为「引导完成→孵化动画→选择系别命名」。需要调整后端 hatch 接口逻辑。

## 五、关键设计决策

### 5.1 引导期间项目创建
- **方案 A（推荐）**：引导过程中允许创建真实项目，引导即实战。用户完成引导后已有第一个项目
- **方案 B**：引导使用虚拟/示例项目，引导结束后用户自行创建。体验不够连贯
- **选择方案 A**：引导就是真实操作流程

### 5.2 引导跳过机制
- 任何步骤都可以点击「稍后再说」跳过
- 跳过后的精灵蛋仍为 L0，下次登录继续引导
- 跳过时提示：「引导完成后我就可以破壳而出，还能获得奖励哦~」

### 5.3 引导奖励
- 完成引导 = 孵化 = 进入 L1
- 孵化时获得初始 +3 天成长时长（复用现有 claimGuideReward 逻辑）
- 奖励在孵化动画后即时发放

### 5.4 引导状态恢复
- 用户中途关闭页面，下次登录从 `guideStep` 恢复
- 如果用户已在工作台页面，引导通过路由跳转到对应页面继续
- 如果目标页面需要数据（如项目 ID），引导会自动导航到可用页面
