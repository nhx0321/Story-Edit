# Story Edit 上线前最终自检修改报告

更新时间：2026-04-30
工作目录：`E:\Story Edit\工具开发\项目\story-edit`

## 1. 背景与目标

本轮工作不是新增功能，而是围绕上线前收口进行深度自检与修正，重点覆盖：

- 安全止血与高风险接口收口
- 前后端导航、命名、入口一致性修正
- 计费/渠道/上下游转发主链核对
- 旧功能、旧资产、旧语义清理
- 大纲/正文导出能力核查与闭环补齐
- 数据库迁移链风险确认
- 最终验证、回滚基线与遗留风险沉淀

本轮优化遵循核心工作流闭环：

1. 建立项目
2. 与 AI 交流，创建全书故事脉络
3. 设定编辑创建基础设定
4. 文学编辑按卷/单元/章节创建梗概
5. 正文作者依据任务书创作草稿
6. 自检与优化建议
7. 自动修改 / 用户手动修改
8. 定稿
9. 对比草稿与定稿，总结经验
10. 更新经验库
11. 阶段性优化提示词，减少冗余和 token 消耗
12. 全书完成后导出创作内容

## 3. 本轮确认并完成的重点修改

### 3.1 安全止血

#### 1) 密码重置接口不再返回调试令牌
文件：`apps/server/src/services/auth/router.ts`

- 删除 `requestPasswordReset` 对外返回的 `_devToken`
- 保留统一提示：账号存在则提示已发送重置链接
- 避免生产环境通过业务接口泄露调试/敏感信息

#### 2) 历史迁移能力改为管理员受限
文件：`apps/server/src/services/migration/router.ts`

- `getStats` 改为管理员可用
- `migrateUser`、`migrateAll` 改为最高管理员权限
- 收口历史迁移工具，避免作为普通业务入口暴露

#### 3) 根路由下线旧域暴露
文件：`apps/server/src/router.ts`

- 从主路由移除旧 `sprite`、`spriteText`、`withdrawal` 挂载
- 保留仍有现网价值的 `spriteBean` 等现行域
- 实现“先收口入口，再考虑后续深度删除”

### 3.2 计费与渠道主链修正

#### 1) 统一按真实用户角色决定限流与路由层级
文件：

- `apps/server/src/routes/ai-stream-platform.ts`
- `apps/server/src/routes/api-v1.ts`

修正点：

- 不再用“是否有余额”替代用户等级判断
- 平台站内调用、站外 API 调用均改为按 `paid/admin -> vip`、其他 -> `free` 进行处理
- 让限流、渠道选择、权限逻辑口径一致

#### 2) 修复成功请求但 usage 缺失时的漏扣问题
文件：

- `apps/server/src/routes/ai-stream-platform.ts`
- `apps/server/src/routes/api-v1.ts`
- `apps/server/src/services/token-relay/token-billing.ts`

修正点：

- 成功调用但上游未返回 usage 时，不再把请求错误地当成零成本
- 改为使用估算 token 进行兜底结算
- 避免“调用成功但未扣费/错扣费”

#### 3) 修复内部计价单位换算
文件：`apps/server/src/services/token-relay/token-billing.ts`

- 将模型价格到内部余额单位的换算统一到 `UNITS_PER_CENT`
- 避免前后端金额展示与后端扣费尺度不一致

#### 4) 外部 API Key 增加白名单与 IP 限流
文件：`apps/server/src/routes/api-v1.ts`

新增：

- 请求 IP 提取
- API Key 白名单校验
- IP 级限流头与错误返回

效果：

- 站外接口边界更明确
- 平台对外开放能力更接近生产可控状态

#### 5) API Key 撤销权限收紧
文件：`apps/server/src/services/token-relay/api-key-service.ts`

- 撤销动作从仅按 `keyId` 改为按 `keyId + userId`
- 避免越权撤销他人 Key

#### 6) Redis 限流初始化兜底修复
文件：`apps/server/src/services/token-relay/rate-limiter.ts`

- 修复 `redis` lazy init 分支判断问题
- 允许 Redis 不可用时正确回落，而不是卡在错误状态

#### 7) 渠道测试接口继续受最高管理员约束
文件：`apps/server/src/routes/ai-stream-platform.ts`

- `/api/ai/stream/channel-test` 增加管理员等级校验

## 6. 遗留问题与风险说明

### 高优先级遗留

#### 1) 数据库迁移链仍存在结构性风险，但本轮已形成首版治理闭环
现状：

- migration 目录存在重复编号
- `apps/server/src/db/migrations/meta/_journal.json` 与目录状态不完全一致
- snapshot 链不完整
- 已确认大量 orphan migrations 未进入正式 journal
- 已确认部分 orphan SQL 与 `0005_lame_doctor_spectrum.sql` 内容重叠，且存在依赖顺序倒挂
- 这些问题仍然说明：当前历史链不能被当作可安全重放的线性事实

本轮已完成：

- 迁移审计：`docs/release-self-check/迁移链审计与治理建议.md`
- 数据库取证：`docs/release-self-check/迁移链取证结果与对账清单.md`
- schema 差异清单：`docs/release-self-check/数据库现状-vs-schema差异清单.md`
- 首批 append-only reconciliation migrations：
  - `0039_content_fingerprints_reconciliation.sql`
  - `0040_token_numeric_reconciliation.sql`
  - `0041_ai_configs_default_reconciliation.sql`
- 本地 `story_edit` 库执行结果与结构快照：`docs/release-self-check/db-snapshots/*`
- migration 状态矩阵：`docs/release-self-check/migration-文件状态矩阵.md`
- 旧体系表保留策略：`docs/release-self-check/旧体系表保留策略清单.md`

当前结论：

- 本轮不再是“只有判断，没有落地”
- 但也仍不适合在上线前直接重排旧历史、批量改名或补写 `_journal.json`
- 更安全的方式仍是：冻结历史、继续追加 reconciliation migrations、分别验证真实升级路径与干净初始化路径

#### 2) 旧体系表已完成首版保留分组，但旧 sprite 陪伴体系尚未彻底下线
现状：

- 已新增：`docs/release-self-check/旧体系表保留策略清单.md`
- 根路由旧 `sprite` / `spriteText` / `withdrawal` 域已收口
- 但数据库层仍存在分化：
  - `user_sprites`、`sprite_bean_transactions`、`recharge_orders` 仍属运行中
  - `subscriptions`、`subscription_orders`、`withdrawal_requests` 归为冻结保留
  - 一批旧 `sprite` / `sprite-text` 陪伴与文本生产表仅进入候删观察组
- 后端代码、静态资源与文档噪音尚未完全拆除

结论：

- 当前状态适合“发布前止血 + 治理分组”
- 不适合在本轮继续做大面积物理删除
- 后续应按“先拆代码依赖、再备份数据、最后用 append-only migration 下线”的顺序推进

#### 3) 仓库存在大量前置改动
现状：

- `git status` 显示当前工作区有大量既有修改和未跟踪文件
- 并非全部来自本轮

结论：

- 本次发布前审阅、回滚、对比必须以 baseline 快照和本报告为准
- 不应直接把当前所有差异视为本轮变更清单
