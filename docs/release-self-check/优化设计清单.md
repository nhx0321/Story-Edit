# Story Edit 核心工作流优化设计清单

更新时间：2026-04-30

本清单只记录“不影响当前核心功能稳定性”的后续优化方向，原则是：

- 不打断当前上线收口
- 不与安全止血、计费主链、迁移治理抢优先级
- 优先做低风险、高收益、贴近写作闭环的改进

## 1. 优化目标

围绕当前主工作流：

建立项目 → AI共创故事脉络 → 设定编辑 → 分层梗概 → 正文创作 → 自检/修改 → 定稿 → 经验沉淀 → 导出

后续优化重点应集中在三件事：

1. 让用户始终知道“下一步去哪”
2. 让 AI 与人工协作切换更顺滑
3. 让阶段性产物沉淀、复用、导出更自然

## 2. 按工作流阶段拆解的优化建议

### 2.1 项目建立阶段

#### 建议 1：建立项目时直接提供“创作模式引导”
目标：减少新用户建完项目后不知道先做什么的问题。

建议内容：

- 新建项目成功后展示清晰的 3 条起步路径：
  - 先搭故事脉络
  - 先录入基础设定
  - 导入已有素材继续创作
- 根据项目类型自动推荐默认路径

价值：

- 提升首次使用成功率
- 减少项目创建后的空白页停滞

#### 建议 2：项目首页加入“当前阶段卡片”
目标：把项目状态从静态页面变成流程导航页。

建议内容：

- 展示当前项目处于：脉络 / 设定 / 梗概 / 正文 / 定稿 / 导出 的哪个阶段
- 给出对应 CTA
- 未完成阶段使用清晰待办样式

### 2.2 AI 共创故事脉络阶段

#### 建议 3：故事脉络产出结构化沉淀
目标：避免用户和 AI 多轮对话后，核心结论散在聊天记录里。

建议内容：

- 在故事脉络页增加“一键沉淀为项目主线摘要”能力
- 输出固定结构：主题、主冲突、角色群像、篇幅规划、卷级方向
- 允许后续编辑引用该摘要作为高层约束

价值：

- 降低后续设定和梗概阶段的上下文漂移
- 减少重复提示词与重复讲述成本

### 2.3 设定编辑阶段

#### 建议 4：设定项增加“引用影响范围”
目标：让设定不再只是静态条目，而成为后续创作约束源。

建议内容：

- 设定条目展示是否已被卷/单元/章节/正文引用
- 修改设定时提示可能影响的下游内容
- 支持把关键设定标为“硬约束”

价值：

- 减少设定变更后的一致性问题
- 为后续自动自检提供依据

### 2.4 梗概分层阶段

#### 建议 5：卷/单元/章节创建时增加“继承链提示”
目标：帮助文学编辑明确当前层级该写到什么粒度。

建议内容：

- 在卷级提示“负责阶段目标和情节走向”
- 在单元级提示“负责承上启下和局部冲突”
- 在章节级提示“负责场景任务、角色行动、信息推进”
- 自动展示上级摘要，减少来回切页

#### 建议 6：章节任务书单独显式化
目标：让正文作者拿到的是可执行任务，而不是散落信息。

建议内容：

- 把章节梗概、角色目标、必须保留信息、语气/风格要求整合为任务书视图
- 正文工作台默认显示“任务书摘要”折叠面板

价值：

- 强化“梗概 → 草稿”衔接
- 降低正文偏题概率

### 2.5 正文创作阶段

#### 建议 7：正文工作台增加“写作闭环状态条”
目标：减少作者在草稿、自检、修改、定稿之间来回摸索。

建议内容：

- 头部展示：草稿中 / 已自检 / 已修改 / 待定稿 / 已定稿
- 每个阶段给出下一步动作
- 自动记录最近一次自检和最近一次修改来源

#### 建议 8：自检结果和修改建议做分层展示
目标：避免建议过多导致作者无法执行。

建议内容：

- 分为必须修、建议修、风格优化三层
- 支持一键仅应用某一类建议
- 支持用户逐条确认而不是整批接受

#### 建议 9：草稿与定稿对比页可视化增强
目标：把经验总结建立在真实差异上。

建议内容：

- 高亮新增、删除、改写段落
- 标记哪些修改来自 AI、哪些来自用户
- 允许从差异中直接生成经验总结草稿

### 2.6 经验库与提示词优化阶段

#### 建议 10：经验沉淀与章节结果自动联动
目标：减少“总结经验”成为额外负担。

建议内容：

- 定稿后自动生成候选经验条目
- 标记经验来自哪一章、哪次修改、解决了什么问题
- 用户确认后再纳入经验库

#### 建议 11：提示词冗余分析面板
目标：响应用户提出的“减少冗余和 token 消耗”。

建议内容：

- 分析当前工作流中重复出现的系统提示、设定摘要、经验提示
- 提示哪些内容可合并、复用、抽成短模板
- 展示优化前后预估 token 差异

价值：

- 直接服务平台成本控制
- 也能改善长上下文模型响应速度

### 2.7 导出阶段

#### 建议 12：导出前增加“导出校验单”
目标：让用户导出前确认作品状态完整。

建议内容：

- 检查是否存在未定稿章节
- 检查是否有空章节标题、空正文、未处理自检结果
- 提供“仅导出已定稿章节”选项

#### 建议 13：导出命名模板可配置
目标：方便用户直接交付或归档。

建议内容：

- 提供命名模板：项目名 / 卷名 / 章节序号 / 导出日期
- 支持一键导出“全书总文档 + 分章文件夹”

## 3. 页面与信息架构层优化

### 3.1 `settings` 与 `ai-config` 边界继续收紧
建议：

- `settings` 只保留账号、资料、系统偏好、精灵豆相关
- `ai-config` 只保留余额、充值、模型、API Keys、站内/站外统计
- 所有余额/模型/Token 消费 CTA 都统一回到 `ai-config`

### 3.2 管理后台信息架构优化
建议：

- 将后台按“用户/模板/渠道/定价/统计/历史工具”分组
- 历史工具统一标记“内部工具”或“仅运维使用”
- 对高风险动作增加二次确认和结果反馈

### 3.3 项目内导航进一步贴近创作流程
建议：

- 项目侧边栏按创作阶段排序，而不是按技术模块排序
- 将“设定、梗概、正文、自检、经验、导出”组织为连续链路
- 降低用户需要理解内部术语的门槛

## 4. 商业主链优化建议

### 建议 14：渠道与计费异常可观测性增强
建议内容：

- 增加预扣、退款、最终结算的链路日志视图
- 后台可按用户、模型、渠道筛选异常请求
- 单独标记“成功但 usage 缺失、走估算扣费”的调用

### 建议 15：用户侧消费解释更透明
建议内容：

- 在站内/站外用量统计中增加“本次扣费如何计算”说明
- 展示模型价格、输入 token、输出 token、最终扣费
- 减少用户对余额变化的不信任感

## 5. 优先级建议

### P0：不影响当前稳定性但收益最高
- 项目首页当前阶段卡片
- 章节任务书显式化
- 正文工作台状态条
- 导出前校验单
- 提示词冗余分析面板

### P1：中期可做
- 设定引用影响范围
- 草稿/定稿差异自动生成经验条目
- 管理后台高风险工具二次确认
- 用户侧消费解释增强

### P2：后续治理类
- 更深的后台信息架构调整
- 更系统的工作流引导重构
- 渠道异常审计仪表盘

## 6. 结论

本轮已经把“导出作品”补进核心写作闭环，但后续真正值得持续投入的方向，不是继续堆页面，而是：

- 让每个创作阶段的输入和输出更清楚
- 让 AI 建议更可执行
- 让经验沉淀更自动化
- 让提示词与 token 成本更可控

这四点决定了 Story Edit 是否能从“功能可用”继续走向“工作流真正高效”。
