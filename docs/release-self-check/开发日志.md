# Story Edit 本轮开发日志

更新时间：2026-05-01
范围：上线前深度自检优化 / 发布前收口

## 1. 本轮目标

围绕发布前收口，完成以下工作：

- 保存当前可回退快照
- 修正生产风险暴露
- 收敛前后端导航、命名、入口边界
- 核对并修复计费/渠道主链关键问题
- 审核旧版导出功能并补齐写作闭环
- 清理无主旧资产与误导性残留
- 执行统一验证并沉淀报告

## 2. 关键过程记录

### 阶段 A：建立基线快照
- 在 `docs/release-self-check/baseline/` 下保存当前状态、差异、迁移目录和关键页面清单
- 明确当前仓库存在大量历史改动，后续必须依赖 baseline 定位本轮变更

### 阶段 B：安全止血
- 去除密码重置接口 `_devToken` 暴露
- 收紧迁移路由权限，限制为管理员/最高管理员
- 从根路由撤下旧 `sprite` / `spriteText` / `withdrawal` 域入口

### 阶段 C：计费与渠道主链核查
- 校正按真实用户角色进行限流和渠道层级判断
- 修复成功请求但 usage 缺失时的漏扣问题
- 修复内部金额单位换算口径
- 增加外部 API Key 的 IP 白名单和 IP 限流
- 收紧 API Key 撤销权限
- 修复 Redis lazy init 限流兜底逻辑
- 强化渠道测试接口的管理员限制

### 阶段 D：前端 IA 与旧语义收敛
- 修正 `ai-config` 余额、统计、消费入口指向
- 把历史订阅页重定向到充值页
- 后台历史迁移页改为内部工具语义
- 重复管理员计费入口改为跳转到统一收入页

### 阶段 E：导出功能补齐
- 审计旧版本导出能力，确认并非完全缺失，而是入口与模式表达不足
- 将“导出大纲”调整为“导出作品”
- 明确大纲导出、正文导出、完整导出三种模式
- 移除未完成的 PDF 对外入口
- 在正文工作台加入“导出正文”按钮
- 让导出能力真正进入创作主工作流

### 阶段 F：旧资产与类型修复
- 删除无主测试图片与旧精灵脚本
- 保留仍被后端依赖的 sprites 资源目录
- 修复 `cleanup.ts` 删除结果类型问题
- 修复 Anthropic/OpenAI 兼容适配器错误响应类型问题
- 修复渠道测试路由变量名错误

### 阶段 G：数据库治理收口
- 在 `schema.ts` 中正式接管 `content_fingerprints`
- 新增 `0039 / 0040 / 0041` 三条 append-only reconciliation migrations
- 在本地 `story_edit` 库执行首批 reconciliation migrations
- 导出 `db-snapshots/` 结构快照（tables / columns / indexes / triggers / enums / foreign_keys）
- 建立 `migration-文件状态矩阵.md`
- 建立 `旧体系表保留策略清单.md`
- 将迁移治理从“风险识别”推进到“本地取证、差异落地、状态沉淀”阶段
