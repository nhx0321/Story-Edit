# 精灵美术资产开发计划

> 风格方向：软陶质感 + 可爱 + 大眼萌（Palworld/Pokemon 风格融合）
> 分辨率：256x256（精灵蛋/低级）、512x512（高级精灵/道具）
> 背景：纯白色（方便后期去底为PNG透明图层）
> 生成策略：资产复用，逐级叠加组件

---

## 一、风格关键词（所有 Prompt 统一使用）

```
soft clay material, subsurface scattering, cute chibi character,
big round sparkling eyes, warm soft lighting, solid white background,
centered composition, game character asset
```

---

## 二、精灵蛋（3种属性 · 256x256）

**核心策略**：同一基础造型，仅换颜色和表面元素纹样

| 属性 | 颜色方案 | 表面元素 |
|------|----------|----------|
| 植物系 | 翠绿渐变 | 叶脉纹理 + 顶部小嫩芽 |
| 动物系 | 暖橙渐变 | 爪印纹理 + 小耳朵凸起 |
| 元素系 | 浅蓝半透明 | 螺旋风纹 + 微光粒子 |

**Prompt 模板：**
```
cute {species} sprite egg, soft clay material, subsurface scattering,
{sphere with gradient color}, {surface texture detail}, {top element},
solid white background, centered, 3D game item asset, chibi scale
```

**资产复用**：3张图共用同一构图和灯光，仅颜色和纹理不同。

---

## 三、狐狸精灵（动物系 · 0-9级 · 512x512）

**核心策略**：每级在前一级基础上**增加组件/图层**，不完全重画

### 级别设计方案

| 等级 | 体型 | 新增特征 | 姿态 |
|------|------|----------|------|
| L0 | 最小（蛋刚孵化） | 刚破壳，带蛋壳碎片，大眼呆萌 | 坐地，好奇张望 |
| L1 | 小巧 | 基础造型，橙白配色，大耳朵，短尾巴 | 站立挥手 |
| L2 | 略大 | 尾巴变蓬松，耳尖出现白色簇毛 | 站立，双手叉腰 |
| L3 | 成长 | 额头出现火焰纹标记，尾巴更蓬松 | 站立，尾巴翘起 |
| L4 | 成熟 | 前爪出现白色"手套"，后爪变深色 | 站立，自信姿态 |
| L5 | 进阶 | 颈部出现小围巾状毛领 | 站立，微微歪头 |
| L6 | 高级 | 尾巴出现金色尖端，耳尖金色装饰 | 站立，优雅姿态 |
| L7 | 精英 | 身体周围微光粒子环绕 | 悬浮，披风状尾巴 |
| L8 | 大师 | 额头火焰纹发光，周身光环 | 悬浮，王者姿态 |
| L9 | 终极 | 全身金光+火焰粒子+光环+翅膀轮廓 | 悬浮，华丽姿态 |

**资产复用策略**：
- L1-L4：同一基础模型，逐步添加纹理（尾巴蓬松度、额纹、手套）
- L5-L6：在 L4 基础上叠加毛领、金色装饰图层
- L7-L9：在 L6 基础上叠加光效粒子/光环图层（可复用光效资源）

**Prompt 递增示例：**
```
L1: ...baby fox, orange white fur, big round eyes, short fluffy tail, standing waving one paw...
L2: L1 prompt + "fluffier tail, white ear tufts"...
L3: L2 prompt + "flame mark on forehead, tail held high"...
...
```

## 四、向日葵精灵（植物系 · 0-9级 · 512x512）

**核心策略**：同样逐级叠加，花盘大小/花瓣层数/装饰递增

| 等级 | 体型 | 新增特征 |
|------|------|----------|
| L0 | 最小 | 刚破壳，小花苞形态，带蛋壳碎片 |
| L1 | 小巧 | 单层黄色花瓣，棕色花盘脸，绿叶手，茎干腿 |
| L2 | 略大 | 花瓣变饱满，出现第二层小花瓣 |
| L3 | 成长 | 花盘出现微笑表情，绿叶变大 |
| L4 | 成熟 | 茎干变粗，出现藤蔓缠绕装饰 |
| L5 | 进阶 | 花瓣边缘出现金色光泽 |
| L6 | 高级 | 花盘周围出现花粉粒子 |
| L7 | 精英 | 茎干出现木质化纹理，顶部小叶片 |
| L8 | 大师 | 花盘发光，周身花粉光环 |
| L9 | 终极 | 多层花瓣+光环+光翼（向日葵花瓣形翅膀） |

---

## 五、风灵精灵（元素系 · 0-9级 · 512x512）

**核心策略**：球体基础，逐步增加透明度变化/风纹/粒子

| 等级 | 体型 | 新增特征 |
|------|------|----------|
| L0 | 最小 | 半透明小光球，带蛋壳碎片 |
| L1 | 小巧 | 半透明蓝色球体，小云手小云脚，大眼 |
| L2 | 略大 | 球体表面出现螺旋风纹 |
| L3 | 成长 | 云手脚变大，出现飘带状尾巴 |
| L4 | 成熟 | 球体内部出现光核 |
| L5 | 进阶 | 周身出现小旋风粒子 |
| L6 | 高级 | 球体更透明，内部光核更亮 |
| L7 | 精英 | 周身旋风粒子变多，形成环状 |
| L8 | 大师 | 光核扩大为光环，球体半透明化 |
| L9 | 终极 | 光环+风翼（云朵形翅膀）+粒子爆发 |

## 六、道具静态图（2D简约风格 · 256x256）

**风格**：扁平化设计 + 轻微渐变 + 白色描边，适合UI图标

| 道具 | 描述 | Prompt关键词 |
|------|------|-------------|
| 经验加速器 | 紫色沙漏，金色简约边框，小翅膀 | `flat 2D icon, purple hourglass, gold frame, small wings, white outline, mobile game UI` |
| 精灵豆 | 绿色椭圆豆，嫩芽，轻微发光 | `flat 2D icon, green bean, sprout, subtle glow, white outline, mobile game UI` |
| 生长药剂 | 绿色药水，向日葵标志 | `flat 2D icon, green potion bottle, sunflower label, white outline, mobile game UI` |

**资产复用**：三个道具共用同一尺寸、描边风格、光影角度。

## 七、动画规划（静态图确认后执行）

### 7.1 精灵蛋动画（3个共用一套动画）
- **孵化前**：轻微上下浮动 + 表面微光闪烁
- **孵化中**：抖动 → 裂开 → 精灵跳出 → 蛋壳碎片消散
- **动画资源**：1套动画模板 × 3种颜色变体

### 7.2 精灵日常动画（每个精灵1套，10级共用）
- **待机**：呼吸起伏 + 尾巴/装饰物微动 + 眨眼
- **互动**：挥手 / 歪头 / 跳跃（可随机触发）
- **策略**：每级精灵使用同一套骨骼动画，仅替换外观图层

### 7.3 精灵升级动画
- **升级瞬间**：光芒包裹 → 体型增大 → 新特征出现 → 光环消散
- **策略**：通用升级动画模板，不同精灵替换粒子颜色

### 7.4 道具动画
- **使用动画**：道具发光 → 飞向精灵 → 消散效果
- **获得动画**：从屏幕上方掉落 → 弹跳 → 进入背包

## 八、执行顺序

### Phase 1：静态图生成
1. ✅ 风格确认（狐狸E/F、向日葵、风灵、道具简约）
2. ⏳ 精灵蛋 × 3（256x256）
3. ⏳ 狐狸精灵 0-9级（512x512）
4. ⏳ 向日葵精灵 0-9级（512x512）
5. ⏳ 风灵精灵 0-9级（512x512）
6. ⏳ 道具 × 3（256x256）

### Phase 2：图片后处理
7. 批量去白色背景 → PNG透明图层
8. 压缩优化（TinyPNG / ImageMagick）
9. 命名规范化（`fox_L1.png`, `sunflower_L3.png` 等）

### Phase 3：动画制作
10. 精灵蛋孵化动画（Lottie / CSS / Canvas）
11. 精灵日常待机动画
12. 精灵升级动画
13. 道具使用/获得动画

### Phase 4：集成到前端
14. 精灵管理页面显示形象（替代emoji）
15. 精灵工作台动画
16. 道具面板图标更新
